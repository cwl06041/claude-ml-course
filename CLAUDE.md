# 仓库简介

这是个人 ML 学习项目，三份手把手讲解的 HTML 教程，全部由 Claude Code 配合用户互动迭代生成。

## 文件结构

| 文件 | 内容 |
|---|---|
| `linear-regression-course.html` | 线性回归完整课程（80KB+，9 个章节：数学预热 → 模型 → 损失 → 互动练习 → 梯度下降 → 超参数 → 编程练习 → 测试 → 后续） |
| `logistic-regression-course.html` | 逻辑回归完整课程（35min 体量，含 sigmoid、对数损失、正则化、二维决策边界互动） |
| `math-viz.html` | 独立数学可视化合集（V 形损失、sigmoid、对数损失爆炸、高斯分布等） |

每个 HTML 是**单文件**，双击浏览器打开即可工作，包含 KaTeX 公式 + 内嵌 SVG 图表 + 交互滑块 + Prism 代码高亮（都从 CDN 加载，需要联网首次加载）。

## 用户背景

- 大学本科数学基础，但很多概念已遗忘（导数、矩阵、概率分布等需要回忆但能跟上）
- 想理解 ML 的"**为什么**"，不仅是"怎么做"
- 主线场景：**房价预测**（线性）、**学生考试通过预测**（逻辑）

## 工作原则（重要 — 改文件前必须遵守）

### 1. 事实为本，不得编造（最高优先级）
- 写"预期输出"必须**真跑代码**验证。曾因凭印象写错 sklearn 输出（b=7.45 错的，真值 b=-5.45）导致整段误导
- 涉及具体数字（命令输出、最优解、收敛步数）一律用 `Bash` + `python3` 跑代码确认
- 不确定的功能/URL/库行为，明说"未验证"，不要硬写

### 2. 渐进性原则（用户多次强调）
不要在课程里提前引用未介绍的概念。三层分级：

| 类型 | 例子 | 处理 |
|---|---|---|
| 代码/工具/库 | `np.sum`、sklearn、numpy | 第⑦节"编程练习"之前**严格禁止** |
| 核心算法名词 | 梯度下降、正则化、softmax | 可作为"后面要讲的 X"提一下 |
| 基础数学概念 | 求导、向量、内积、梯度 | 必须先在⓪节"数学预热"覆盖 |

### 3. 倒金字塔回答
- 输出第一句必须是核心结论/推荐
- 决策类问题：第一句"建议 A，因为 X"，下面再展开 A/B/C 对比
- 推理放 thinking 块里，输出阶段答案已成型

### 4. 视角切换显式说明
当图表的横纵轴/坐标系含义改变时（如：数据视角 vs 参数视角），**必须用对比表显式说明**。不假设读者能自己脑补"现在看的是什么"。

### 5. SVG > ASCII art
数学图全部用 SVG（KaTeX 公式 + 真实坐标轴 + 精确曲线）。ASCII art 视觉降级，仅在极少情况下用。

### 6. `<details>` 控制深度
不是所有人都需要细节。推导、矩阵形式、可选证明放可折叠块，标题说清适用对象（"如果你学过矩阵+求逆"）。

### 7. 不要把对话推理塞进 HTML 正文
对话里"我和用户来回推演、否定备选"的过程，**不要写进文档**。只写"最终成型的内容"。用户讨厌这点。

## 已经踩过的实战坑（避坑指南）

### 坑 1：学习率必须验证
- 线性回归用未标准化的房价数据（X ∈ [50, 140]），Hessian ≈ 19700
- **安全学习率 lr ≤ 0.0001**
- 任何新加 GD 演示：先用 Python 跑一遍确认收敛，不要凭直觉设 lr
- 曾经默认 lr=0.0005 → 走一步就发散到 60 亿损失

### 坑 2：可视化条件数问题
- 房价数据 $(\Sigma x^2/n) : 1 = 9850:1$，等高线极扁，不适合画"漂亮椭圆"
- 画 2D 损失曲面时：用**平衡的合成数据** + 在文字中说明原因（顺便讲条件数概念）

### 坑 3：初始可见性
- 若初始参数 $(w=0, b=0)$ 让直线刚好等于轴值，会误以为"没渲染"
- 要么扩展 yMin 到负值让线明显可见，要么明确标注"初始位置"

### 坑 4：tangent vs chord 画错
- 在 1D 损失曲线上画 GD 演示时，"连续两个 step 之间的连线"是**弦**，不是导数
- 导数 = **切线**，要单独画
- 弦画在抛物线内侧（凸函数性质），切线画在曲线外侧或相切

### 坑 5：编造预期输出
- 写 sklearn / numpy 训练代码时不要凭直觉填 "expected output"
- 用 Bash + python3 真跑一次拿真实数字

## 工作流（修改时遵循）

1. **修改 HTML 文件**（用 Edit 工具，不用 Write 大重写）
2. **可视化验证**：用 `mcp__Claude_Preview__preview_*` 工具开 server 截图确认
3. **数字验证**：用 Bash 跑 `python3 -c "..."` 确认输出
4. **告诉用户刷新浏览器**

## 已知工具配置
