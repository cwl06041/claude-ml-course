⓪ 数学预热

线性回归用到的数学其实不多，但我们先把要用到的"零件"摆出来。这一章可以快速翻翻，遇到生疏处再细看。

0.1 函数：把输入变成输出的盒子

场景：你想根据房屋面积估算价格。每给一个面积，给一个价格。这就是个函数。

记号 $y = f(x)$：

  • $x$ 叫自变量（你能控制 / 输入的，比如面积）
  • $y$ 叫因变量（被决定 / 输出的，比如价格）
  • $f$ 是把 $x$ 变成 $y$ 的规则

线性回归里我们假设这个规则长这样：$f(x) = wx + b$ —— 直线。

0.2 斜率与截距：直线的两个参数

  • $w$ 斜率：x 每增加 1，y 增加多少。陡不陡。
  • $b$ 截距：x = 0 时 y 是多少。直线在 y 轴上的位置。
直线"长什么样"由 $w, b$ 完全决定。整个线性回归的任务，就是找到最好的 $w, b$。

0.3 导数：瞬时变化率

导数 $f'(x)$ 回答这个问题：在 $x$ 这个点，函数变化得多快、往哪边变？

几何上：导数 = 函数曲线在该点的切线斜率。

函数导数几何意义
$f(x) = c$（常数）$0$水平线，不变化
$f(x) = wx + b$$w$直线，处处斜率相同
$f(x) = x^2$$2x$抛物线，离原点越远越陡
$f(x) = (x-3)^2$$2(x-3)$$x=3$ 处斜率为 0（最低点）

求函数最值的"三步法"（机器学习训练的数学基础）

给定一个函数 $f(x)$，要找它的最小值（或最大值），标准方法是：

  1. 对 x 求导得到 $f'(x)$
  2. 令 $f'(x) = 0$，得到一个关于 x 的方程
  3. 解这个方程，得到的 x 就是候选极值点

为什么这样可行：函数在山顶/谷底处的切线是水平的（变到平了才能转折），切线斜率就是导数。所以"极值点 ⇔ 导数 = 0"。

例子：找 $f(x) = (x-3)^2$ 的最小值。

  1. $f'(x) = 2(x-3)$
  2. 令 $2(x-3) = 0$
  3. 解得 $x = 3$，对应最小值 $f(3) = 0$ ✓

多变量怎么办：变成"每个方向的偏导都 = 0"——比如有两个变量 $w, b$，就要 $\dfrac{\partial f}{\partial w} = 0$ 和 $\dfrac{\partial f}{\partial b} = 0$ 同时成立（否则那个方向还能继续下降）。这给出两个方程，两个未知数，可以联立解。

注意一个坑：导数=0 的点不一定就是最小值——也可能是最大值或鞍点。但对于凸函数（碗形），导数=0 的点一定就是唯一的全局最小值。我们的损失函数恰好是凸的，所以这招屡试不爽。

这是机器学习训练的数学底子。后面的"正规方程"就是把这一招套到损失函数上——把损失对 $w, b$ 的偏导都设成 0，联立解出最优 $(w^*, b^*)$。

0.4 偏导数：多变量时只动一个

真实场景里房价不只取决于面积，还有房龄、地段等。函数变成 $f(x_1, x_2, \ldots)$。这时候问"变化率"就要指明变哪一个。

记号 $\dfrac{\partial f}{\partial x_1}$：把 $x_2, x_3, \ldots$ 全当成常数，只对 $x_1$ 求导。

例子：$f(w, b) = (wx + b - y)^2$，对 $w$ 求偏导

把 $b, x, y$ 都当常数，只对 $w$ 求导。用链式法则：

$$\frac{\partial f}{\partial w} = 2(wx+b-y) \cdot x$$

外层平方 → 系数 2 拉下来；内层 $wx+b-y$ 对 $w$ 求导 = $x$。

0.5 向量与内积：多元情况的简洁写法

有 $d$ 个特征时，写 $w_1 x_1 + w_2 x_2 + \cdots + w_d x_d + b$ 太啰嗦。用向量：

$$\mathbf{w} = \begin{pmatrix} w_1 \\ w_2 \\ \vdots \\ w_d \end{pmatrix}, \quad \mathbf{x} = \begin{pmatrix} x_1 \\ x_2 \\ \vdots \\ x_d \end{pmatrix}$$

内积（也叫点积）：$\mathbf{w}^\top \mathbf{x} = \sum_i w_i x_i$。所以模型可以紧凑地写成：

$$\hat{y} = \mathbf{w}^\top \mathbf{x} + b$$

这是为了少打字、少出错，本质和写出来的一长串完全一样。看到 $\mathbf{w}^\top\mathbf{x}$ 就是"对应位置相乘再相加"。

0.6 向量求导（仅在读 ②.4 矩阵形式推导时需要）

线性回归用矩阵形式重写后会出现 $\nabla_{\mathbf{w}}(\mathbf{a}^\top \mathbf{w})$ 这种记号——这是"标量函数对向量求导"，也叫"梯度"。如果你不打算读矩阵推导，可以跳过这一节。

定义：梯度就是"偏导打包"

$f(\mathbf{w})$ 接受一个向量、输出一个数（标量函数）。它对 $\mathbf{w}$ 的"导数"$\nabla_{\mathbf{w}} f$ 就是把所有偏导数堆成一个向量（形状和 $\mathbf{w}$ 一样）：

$$\nabla_{\mathbf{w}} f = \begin{pmatrix} \partial f / \partial w_1 \\ \partial f / \partial w_2 \\ \vdots \\ \partial f / \partial w_d \end{pmatrix}$$

本质上和 ⓪.4 节的偏导一模一样，只是把若干偏导打包成向量方便书写。在机器学习里这个向量就叫"梯度"（gradient）——你以后看到 $\nabla L$ 都是这个意思。

两个常用公式（②.4 矩阵推导只用到这两个）

公式 1（线性式的梯度）：

$$\nabla_{\mathbf{w}}(\mathbf{a}^\top \mathbf{w}) = \mathbf{a}$$

类比单变量：$(ax)' = a$。$\mathbf{a}^\top\mathbf{w}$ 是 $\mathbf{w}$ 的"线性项"，对 $\mathbf{w}$ 的梯度就是系数向量 $\mathbf{a}$。

看一眼推导（一行就完）

$\mathbf{a}^\top\mathbf{w} = a_1 w_1 + a_2 w_2 + \cdots + a_d w_d$（一个数）。

对 $w_k$ 求偏导：只有 $a_k w_k$ 这一项含 $w_k$，结果是 $a_k$。

把 $d$ 个偏导堆起来 = $(a_1, a_2, \ldots, a_d)^\top = \mathbf{a}$ ✓

公式 2（二次型的梯度，$A$ 对称时）：

$$\nabla_{\mathbf{w}}(\mathbf{w}^\top A \mathbf{w}) = 2 A \mathbf{w}$$

类比单变量：$(a x^2)' = 2 a x$。$\mathbf{w}^\top A \mathbf{w}$ 是 $\mathbf{w}$ 的"二次项"，梯度是 $2 A \mathbf{w}$。

看一眼推导

展开：$\mathbf{w}^\top A \mathbf{w} = \sum_{i,j} A_{ij} w_i w_j$（双重求和，遍历矩阵所有元素）。

对 $w_k$ 求偏导。在所有 $A_{ij} w_i w_j$ 项里，含 $w_k$ 的有两种：

  • $i = k$（$j$ 任意）：这一项是 $A_{kj} w_k w_j$，对 $w_k$ 求导得 $A_{kj} w_j$。$j$ 求和：$\sum_j A_{kj} w_j$
  • $j = k$（$i$ 任意）：这一项是 $A_{ik} w_i w_k$，对 $w_k$ 求导得 $A_{ik} w_i$。$i$ 求和：$\sum_i A_{ik} w_i$

合起来：$\frac{\partial}{\partial w_k}(\mathbf{w}^\top A\mathbf{w}) = \sum_j A_{kj} w_j + \sum_i A_{ik} w_i$。

当 $A$ 对称（$A_{ij} = A_{ji}$）时两项相等：$= 2 \sum_j A_{kj} w_j = 2 (A\mathbf{w})_k$。

把所有偏导堆起来 = $2 A \mathbf{w}$ ✓

⚙️ $A$ 不对称时是 $(A + A^\top)\mathbf{w}$，但我们在线性回归里只会遇到 $X^\top X$ 这种天生对称的矩阵，所以对称版本够用。

为什么 $X^\top X$ 一定对称？因为 $(X^\top X)^\top = X^\top (X^\top)^\top = X^\top X$。任何形如 $A^\top A$ 的矩阵都自动对称。这就是为什么 ②.4 节矩阵推导里 $\mathbf{w}^\top X^\top X \mathbf{w}$ 能直接套公式 2。
这两个公式是线性回归矩阵推导的全部数学武器。学过它们之后，整个推导只要三行：损失展开 → 套公式求梯度 → 令梯度=0 → 移项得 $\mathbf{w}^* = (X^\top X)^{-1} X^\top \mathbf{y}$。

① 线性回归是什么

真实场景：你拿到一份 200 套二手房的数据，每套记录了面积、房龄、距地铁距离和成交价。 现在新挂出一套房，知道这三个特征，问：合理价格大概是多少？

1.1 直觉：找一条"贴合数据"的直线

先简化只看一个特征：面积。把已知数据画在坐标系里：

这些点大致沿着一个方向铺开 —— 面积越大、价格越高。我们想找一条能代表这种趋势的直线，新房子来了，就用这条直线预测它的价格。

1.2 数学：模型公式

一元（一个特征）：

$$\hat{y} = wx + b$$

  • $x$：面积
  • $\hat{y}$：模型预测的价格（戴帽子的 $y$ 表示"预测值"，区别于真实值 $y$）
  • $w, b$：等待我们找出来的参数

多元（多个特征）：

$$\hat{y} = w_1 x_1 + w_2 x_2 + w_3 x_3 + b = \mathbf{w}^\top \mathbf{x} + b$$

  • $x_1$ 面积，$x_2$ 房龄，$x_3$ 距地铁距离
  • $w_1, w_2, w_3$ 分别是每个特征的权重（你贡献多少价格）
  • $b$ 是"底价"
"线性"二字的含义：每个特征对结果的影响是独立且按比例的。 面积每多 1 m²，价格固定多 $w_1$ 元，不管房龄是多少。 这是个简化假设，现实里可能不完全成立（比如学区房面积大可能溢价更高）， 但它是个不错的起点 —— 简单、可解释、好优化。

1.3 几何直觉

  • 1 个特征 → 找一条直线
  • 2 个特征 → 找一个平面
  • d 个特征 → 找一个 d 维超平面（看不见了，但数学一样）

1.4 多特征是怎么"独立工作"的

当模型有多个特征时，每个特征都独立贡献对价格的预测——把每个特征单独看，它和价格仍然是直线关系。下面用 15 套房子的数据，分别看面积、房龄、距地铁三个特征各自和价格的关系：

面积越大价格越高（正斜率）；房龄越大价格越低（负斜率）；距地铁越远价格越低（负斜率）。模型 $\hat{y} = w_1 \cdot 面积 + w_2 \cdot 房龄 + w_3 \cdot 距地铁 + b$ 同时学到这三个斜率。

1.5 用线性还是不用？

场景线性合适吗原因
面积 → 房价✅ 不错大体线性
广告投放 → 销售额⚠️ 短期可以长期有边际递减
药物剂量 → 疗效❌ 不合适过量反而有害（U 型）
图像识别❌ 完全不行像素到类别的关系极度非线性
线性模型的价值不在"能解决所有问题"，而在"是理解一切的起点"——逻辑回归、神经网络、深度学习都是在它的骨架上加东西。

② 损失：什么叫"好的直线"

问题：直线有无数条，哪一条最好？我们需要一个量化的标准，叫做损失函数—— 它给每条直线打分，分数越低越好。

2.1 直觉：误差应该被惩罚

对每个数据点 $(x_i, y_i)$：

  • 模型预测 $\hat{y}_i = w x_i + b$
  • 真实值 $y_i$
  • 误差 $\varepsilon_i = y_i - \hat{y}_i$（也叫残差）

我们希望所有点的残差都尽量小。但残差有正有负，直接加起来会互相抵消，得想个办法。

2.2 三种候选的损失函数

名字公式性质
残差求和$\sum \varepsilon_i$❌ 正负抵消，无意义
绝对值损失 (MAE)$\frac{1}{n}\sum |\varepsilon_i|$⚠️ 不可导（V 字尖）
平方损失 (MSE)$\frac{1}{n}\sum \varepsilon_i^2$✅ 处处可导、有解析解、对应高斯 MLE

2.3 平方损失 (MSE)

$$L(w, b) = \frac{1}{n} \sum_{i=1}^{n} (y_i - (w x_i + b))^2$$

🎯 一个容易卡住的认知点：x, y, w, b 的角色对调（强烈建议展开）

从初等数学过来，我们习惯把 $x$ 当自变量、$y$ 当因变量。看到 $L(w, b) = \frac{1}{n}\sum(y_i - wx_i - b)^2$ 里同时出现 $w, b, x, y$ 还带个求和符号，会觉得别扭——到底谁是变量？

关键切换：在训练阶段，$x_i, y_i$ 是已知的数据（一堆固定数字），$w, b$ 才是要解的自变量。求和符号只是"把数据点加在一起"。

把损失展开看，所有 $\sum$ 都可以被吸进系数里：

$$L(w, b) = \frac{1}{n}\Big[\big(\textstyle\sum x_i^2\big) w^2 + 2\big(\textstyle\sum x_i\big) wb + nb^2 - 2\big(\textstyle\sum x_iy_i\big) w - 2\big(\textstyle\sum y_i\big) b + \textstyle\sum y_i^2\Big]$$

每个 $\sum$ 都是给定数据下的一个具体数字。用前面 10 套房子的数据算一下：

$$L(w, b) = 9950\,w^2 + 190\,wb + b^2 - 72100\,w - 680\,b + 131060$$

看，一旦数据代进去，损失就只是个 $w, b$ 的二次函数——固定 $b$ 看它，就是个初中你就会的一元二次函数 $Aw^2 + Bw + C$，找最低点令导数=0 即可。

命名陷阱提示：$y = x^2$ 这种碗形函数在数学里叫"凸函数（convex）"，不是"凹"。汉字"凸"字形是往上鼓的（⊥），所以直觉上会想反——觉得碗形该叫"凹"。但数学命名跟字形是反的。判据看二阶导数：$y''>0$ 是凸（碗）、$y''<0$ 是凹（倒碗）。$y=x^2$ 的 $y''=2>0$，所以是凸。
视角$x_i, y_i$ 是$w, b$ 是
高中 $y = wx + b$自变量 / 因变量直线的形状参数（先给定）
ML 训练阶段固定数据（常数）要优化的自变量
ML 预测阶段又变回自变量（新输入）已学好的常数（模型权重）

同一组符号在训练和预测时角色对调，这是机器学习初学最大的认知坎。意识到这点之后：

  • 损失只是个二次函数 → 求导令 = 0 找最低点 = 正规方程的来历
  • 二次函数是凸函数（碗形，convex）→ 损失曲面是个碗 → 梯度下降一定能收敛到碗底 → 能找到全局最小的损失，对应那条最契合数据的直线
  • 所有那些复杂公式只是"求和"形式的紧凑写法，本质是初中代数

那为什么教科书坚持用 $\sum$ 写而不直接展开成具体系数？

  1. 通用性：$\sum x_iy_i$ 不需要知道 $n$，10 个样本和 1000 万个样本是同一个公式
  2. 求导更整齐：对 $\sum$ 求导可以"先对每一项求导再求和"（求和与求导可交换），整个推导一气呵成；展开成具体系数后反而要追踪一堆零散的项

2.4 为什么平方损失？三个理由

注意下面三个理由回答的是不同层次的问题，不是互相重复——

理由它回答的问题层次
① 处处可导"这个损失能不能用算法训练？"工程可行
② 有解析解"这个损失训练快不快？"工程效率
③ 对应 MLE"为什么是平方，不是 4 次方、立方、绝对值？"统计正当性

没有 ③，你只知道"平方损失方便"，但不知道"换其他损失会不会本质上更合理"。MLE 那个推导解释了平方损失的"出身"——是从正态分布密度公式里的 $e^{-\varepsilon^2}$ 自然导出来的，不是有人拍脑袋拍出来的。

① 处处可导 → 梯度下降好用

平方损失的偏导（把 $\hat{y}_i = wx_i + b$ 代回去）：

$$\frac{\partial L}{\partial w} = -\frac{2}{n}\sum_{i=1}^n x_i(y_i - wx_i - b)$$

把求和分配进去看清楚：

$$\frac{\partial L}{\partial w} = \underbrace{\frac{2}{n}\sum x_i^2}_{\text{常数 }A} \cdot w \; + \; \underbrace{\frac{2b}{n}\sum x_i - \frac{2}{n}\sum x_i y_i}_{\text{常数 }B}$$

$x_i, y_i$ 是固定的数据，所以整个偏导就是 $w$ 的一次函数 $Aw + B$，图像是一条直线——处处连续、没有任何跳跃。梯度作为 $w$ 的函数是平滑的，就是这个意思。

绝对值损失的偏导就完全不一样。先看单个样本的：

$$\frac{\partial}{\partial w}|y_i - wx_i - b| = -\text{sign}(y_i - wx_i - b) \cdot x_i$$

$\text{sign}$ 函数只能输出 $-1, 0, +1$。当 $w$ 调到某个值让残差 $y_i - wx_i - b$ 恰好等于 0 时，再稍微动一下 $w$，$\text{sign}$ 就从 $-1$ 突跳到 $+1$。单样本偏导是个阶跃函数（1 个跳跃点）。

把所有 $n$ 个样本加起来，才是真正的损失偏导：

$$\frac{\partial L}{\partial w} = -\frac{1}{n}\sum_{i=1}^n \text{sign}(y_i - wx_i - b) \cdot x_i$$

$n$ 个阶跃函数叠加 → $n$ 个跳跃点的阶梯函数。每个样本会在它自己残差变号的位置贡献一次跳跃，10 个样本 10 个跳跃点，1000 个样本 1000 个跳跃点。

偏导关于 $w$ 长什么样梯度下降的表现
平方损失$Aw+B$，一条直线方向缓变 → 稳步收敛
绝对值损失sign 函数，阶跃在残差=0 附近方向突变 → 容易震荡
梯度下降的每一步靠"沿当前梯度的反方向走一小步"。梯度方向缓变 → 走得稳，平滑下山；梯度方向突变 → 走着走着方向反了，参数在最优点附近反复跨过去再跨回来，难收敛。

② 有解析解（闭式公式）

令偏导 = 0，可以直接解出最优 $w, b$ 的公式（叫"正规方程"）。绝对值损失没有这种公式。

展开看正规方程的推导

核心思路：损失 $L$ 是 $w, b$ 的二次函数（碗形），按 ⓪.3 节的"求最值三步法"——最低点处偏导等于 0（碗底的切线水平）。求最优 $(w^*, b^*)$ ≡ 解"两个偏导都等于 0"这个方程组。这就是正规方程名字的来历。

对 $w$ 求偏导，令这个偏导本身等于 0：

$$\frac{\partial L}{\partial w} = -\frac{2}{n}\sum_{i=1}^n x_i(y_i - wx_i - b) = 0$$

对 $b$ 求偏导，令这个偏导本身也等于 0：

$$\frac{\partial L}{\partial b} = -\frac{2}{n}\sum_{i=1}^n (y_i - wx_i - b) = 0$$

把 $-\frac{2}{n}$ 约掉，求和分配进去，把含 $w, b$ 的项提出来：

$$\big(\textstyle\sum x_i^2\big)\,w + \big(\textstyle\sum x_i\big)\,b = \textstyle\sum x_iy_i$$

$$\big(\textstyle\sum x_i\big)\,w + n\,b = \textstyle\sum y_i$$

每个 $\sum$ 都是数据给定的固定数字。所以这是两个关于 $(w, b)$ 的一次方程——注意 $w, b$ 都是一次方（没有 $w^2$、$wb$ 这种），所以叫"线性方程组"。两个未知数两个一次方程 → 初中代入消元法直接解，得到唯一的 $(w^*, b^*)$。

⚠️ 这一招专属于"线性回归 + 平方损失"这种偏导=0 能直接解出来的模型。逻辑回归、神经网络等更复杂模型的偏导=0 方程根本解不出来 —— 那时候就要换用 ④ 节讲的梯度下降了。

展开看矩阵形式的推导（如果你学过矩阵 + 求逆）

当特征数变多时，写一堆 $\sum$ 会很啰嗦。用矩阵记号能把推导压缩成几行。

记号约定

设 $n$ 个样本、$d$ 个特征。为了把截距 $b$ 也吸进权重向量，给每个样本前面补一个 1：

$$X = \begin{pmatrix} 1 & x_{11} & \cdots & x_{1d} \\ 1 & x_{21} & \cdots & x_{2d} \\ \vdots & \vdots & & \vdots \\ 1 & x_{n1} & \cdots & x_{nd} \end{pmatrix} \in \mathbb{R}^{n\times(d+1)}, \quad \mathbf{y} = \begin{pmatrix} y_1 \\ \vdots \\ y_n \end{pmatrix}, \quad \mathbf{w} = \begin{pmatrix} b \\ w_1 \\ \vdots \\ w_d \end{pmatrix}$$

那么所有预测一次性写出来：$\hat{\mathbf{y}} = X\mathbf{w}$（$n$ 维向量）。

损失的矩阵形式

$$L(\mathbf{w}) = \frac{1}{n}\|\mathbf{y} - X\mathbf{w}\|^2 = \frac{1}{n}(\mathbf{y} - X\mathbf{w})^\top (\mathbf{y} - X\mathbf{w})$$

展开（用 $(\mathbf{a}-\mathbf{b})^\top(\mathbf{a}-\mathbf{b}) = \mathbf{a}^\top\mathbf{a} - 2\mathbf{a}^\top\mathbf{b} + \mathbf{b}^\top\mathbf{b}$）：

$$L(\mathbf{w}) = \frac{1}{n}\big(\mathbf{y}^\top\mathbf{y} - 2\mathbf{y}^\top X\mathbf{w} + \mathbf{w}^\top X^\top X \mathbf{w}\big)$$

对 w 求梯度（向量求导）

用 ⓪.6 节的两个标准公式 $\nabla_{\mathbf{w}}(\mathbf{a}^\top \mathbf{w}) = \mathbf{a}$ 和 $\nabla_{\mathbf{w}}(\mathbf{w}^\top A \mathbf{w}) = 2A\mathbf{w}$（$A$ 对称时）：

$$\nabla_{\mathbf{w}} L = \frac{1}{n}\big(-2X^\top\mathbf{y} + 2X^\top X \mathbf{w}\big) = \frac{2}{n}\big(X^\top X \mathbf{w} - X^\top \mathbf{y}\big)$$

令梯度 = 0，解出 w

$$X^\top X \mathbf{w} = X^\top \mathbf{y}$$

这就叫正规方程（normal equation）。如果 $X^\top X$ 可逆（一般情况下都可逆），两边同乘 $(X^\top X)^{-1}$：

$$\boxed{\mathbf{w}^* = (X^\top X)^{-1} X^\top \mathbf{y}}$$

对应回标量版本

把 $X$ 只有一列（加截距列就是两列）的情况展开，正好是上面那个标量版本的方程组。矩阵形式是多特征版本的紧凑写法，本质完全一样。

⚙️ 这就是 sklearn 的 LinearRegression.fit() 内部实际跑的公式。它对 $X^\top X$ 做矩阵求逆（实际上用更稳定的算法比如 QR 分解或 SVD），一次性算出最优 $\mathbf{w}^*$。$O(d^3)$ 的复杂度也来自这一步——矩阵求逆要做这么多次乘法。

③ 对应"高斯噪声"假设下的最大似然估计 (MLE)

⚠️ 没学过 / 忘了"高斯噪声"先看这里（强烈建议展开）

第一步：什么是"噪声"？

现实里每个测量都不完美。比如同一根 1 米长的杆子，量 10 次得到 0.99 / 1.01 / 1.00 / 1.02 / 0.98 / ... 这些偏差就叫"噪声"。

放到房价场景里：假设房价完全由面积决定，那"真实直线" $y = wx + b$ 是存在的。但你观察到的每个 $y_i$ 都会被一堆我们看不到的小因素（学区、装修、卖家心情、谈判技巧……）合起来推高或压低一点。把这个偏差记作 $\varepsilon_i$：

$$y_i = wx_i + b + \varepsilon_i$$

$\varepsilon_i$ 就是"噪声"——你观察不到、但确实存在的偏差。

第二步：噪声长什么样？正态分布 / 高斯分布

对很多现实场景观察发现，噪声有几个共同特点：

  • 偏差 ≈ 0 的情况最多（大部分时候大致对）
  • 偏差越大越罕见（差 100 万的情况比差 1 万的少得多）
  • 左右对称（高估和低估出现的概率差不多）

这种"钟形"概率分布，数学上叫正态分布，因为是高斯发现并研究的，也叫高斯分布。它的形状：

密度公式（看一眼就行，不用记）：

$$f(\varepsilon) = \frac{1}{\sqrt{2\pi}\sigma} \exp\!\left(-\frac{\varepsilon^2}{2\sigma^2}\right)$$

关键事实：$\varepsilon$ 离 0 越远，密度按 $e^{-\varepsilon^2}$ 急剧下降（指数级地稀薄）。注意里面有个 $\varepsilon^2$ —— 平方让大偏差变得"指数级罕见"。这个平方很快就会和我们的"平方损失"对上。

第三步：什么叫"高斯噪声假设"？

就是说：我们假定上面那个 $\varepsilon_i$ 服从正态分布。在现实中这个假设大体成立 —— 根据"中心极限定理"，很多个独立的小因素加起来，整体结果就近似正态。学区+装修+心情+谈判+市场……一堆小因素叠加，整体偏差自然趋向钟形。所以"高斯噪声"在数据建模里是个非常常见、且通常合理的默认假设。

MLE（最大似然估计）的核心问题：手里有这批数据，哪组 $(w, b)$ "最可能产生" 它们？答案就是让"这批数据出现的总概率"最大的那组。

展开看具体推导（看到平方损失从哪冒出来）

给定参数 $(w, b)$，第 $i$ 条样本的"出现概率密度" = 噪声 $\varepsilon_i = y_i - wx_i - b$ 在正态分布里的密度：

$$P(y_i \mid x_i; w, b) = \frac{1}{\sqrt{2\pi}\sigma} \exp\!\left(-\frac{(y_i - wx_i - b)^2}{2\sigma^2}\right)$$

$n$ 条样本相互独立，总概率（叫"似然" $\mathcal{L}$）= 各概率相乘：

$$\mathcal{L}(w, b) = \prod_{i=1}^n \frac{1}{\sqrt{2\pi}\sigma} \exp\!\left(-\frac{(y_i - wx_i - b)^2}{2\sigma^2}\right)$$

连乘不好算，取对数变连加。把和 $(w, b)$ 无关的常数项捏到一起：

$$\ln \mathcal{L} = -\frac{1}{2\sigma^2}\sum_{i=1}^n (y_i - wx_i - b)^2 \;+\; \text{常数}$$

最大化 $\ln \mathcal{L}$ 等价于最小化 $\sum (y_i - wx_i - b)^2$ —— 这就是平方损失！

平方损失不是有人拍脑袋写的，它有统计学根基：当噪声呈钟形分布时（这在现实中很常见），最小化平方损失就等价于"找到最可能产生这批数据的 $(w, b)$"。
再换个角度：损失函数里的"平方"，其实是从正态分布密度公式 $e^{-\varepsilon^2}$ 里的那个平方继承下来的。

2.5 你以后会在代码/论文里看到的几个简写

本章主角是平方损失。但你以后会在文档、Kaggle、论文里反复看到 L1 / L2 / MAE / MSE / RMSE 这几个名字——它们其实是同一族的不同写法。这一节提前认个脸熟。

L1 / L2 是"族名"

损失函数按"怎么衡量误差"分两大族：

  • L1 损失：用绝对值衡量，$\sum_i |\varepsilon_i|$
  • L2 损失：用平方衡量，$\sum_i \varepsilon_i^2$（本章主角）

MAE / MSE / RMSE 是"具体形式"

名字公式属于哪族什么时候用
MAE 平均绝对误差$\frac{1}{n}\sum |\varepsilon_i|$L1 / N数据有极端离群点不想被绑架时
MSE 均方误差$\frac{1}{n}\sum \varepsilon_i^2$L2 / N训练时用，本章用的就是它
RMSE 均方根误差$\sqrt{\text{MSE}}$L2 派生报告结果时用，单位和 y 一样
记忆要点：L1/L2 是族名，MAE/MSE/RMSE 是具体形式。差别只是"要不要除以 N（求平均）"和"要不要开根号（恢复单位）"。同族内最优 $(w, b)$ 一样——因为除以 N 和开根号都是单调变换，不改变最小化结果。所以训练时哪个方便用哪个；汇报用 RMSE 因为"平均偏差 30 万"比"误差 900 万²"好懂。

L2 vs L1 在离群值场景的差别

"L2 对大误差敏感、L1 对离群值鲁棒"这话很抽象，看图：同一批房价数据故意混进 1 个离群点（"豪宅"：100 m² 卖到 700 万），用 L2/MSE 和 L1/MAE 分别拟合：

L2/MSE 拟合 L1/MAE 拟合 正常数据 离群点
L2 把误差平方 → 大误差被放大 → 模型为了缩小"那个特别大的误差"会被拉向离群值。
L1 把误差取绝对值 → 大小误差一比一计入 → 模型不会为了照顾一个极端点而牺牲多数点。
场景选谁
"大错"代价远大于"小错"（医疗、风控）L2 / MSE
数据里离群值是"坏数据"（录入错、传感器抖动）L1 / MAE
不确定 / 想要稳健默认MAE 或 Huber loss（两者折中：小误差用 L2、大误差用 L1）

④ 梯度下降：让机器自己找最优

等一下：刚才正规方程已经直接算出最优 $(w^*, b^*)$ 了，为什么还要梯度下降？两件事：
① 多数模型没有解析解——偏导=0 解不出来；
② 正规方程在特征多时算不动。
🔍 "解不出来"和"算不动"具体是什么意思（强烈建议展开）

① 为什么有些模型"解不出来"？

"解不出来"不是说"没有最优"，而是没有一行公式能直接算出来。

线性回归（能解）：偏导=0 展开后是

$$\big(\textstyle\sum x_i^2\big) w + \big(\textstyle\sum x_i\big) b = \textstyle\sum x_iy_i$$

$w, b$ 都是一次方（没有 $w^2$、$e^w$ 这种），初中代入消元法直接搞定。

逻辑回归（解不出）：偏导=0 含 sigmoid $\sigma(z) = 1/(1+e^{-z})$，展开后含 $e^{-(wx+b)}$ —— 这是超越方程（含指数函数）。中学代数里没有"如何解含 $e^x$ 的方程"，因为它没有代数解析解。

神经网络（更解不出）：几百万个参数，方程组规模太大，且层之间嵌套非线性变换。理论上有最优，但写不出"解的公式"。

所以：这些模型只能用迭代算法（梯度下降）一步步逼近最优。

② 什么是"矩阵求逆"、为什么算不动？

这里只浅讲，等以后学线性代数会深入。

类比：单个数 5 的"倒数"是 $\frac{1}{5}$，$5 \times \frac{1}{5} = 1$。

矩阵就是一个数表（比如 $d \times d$ 的方阵）。矩阵也有类似的"倒数"概念，叫"逆矩阵" $A^{-1}$，满足 $A \cdot A^{-1} = $ 单位矩阵（相当于数字 1）。

线性回归的正规方程（向量化形式）是：

$$\mathbf{w}^* = (X^\top X)^{-1} X^\top \mathbf{y}$$

里面那个 $(\cdot)^{-1}$ 就是求逆。算一个 $d \times d$ 矩阵的逆，需要做 $O(d^3)$ 次乘法。给点具体数字感受：

特征数 d$d^3$ 操作数大致耗时
101,000瞬间
1001,000,000瞬间
1,00010 亿几秒
10,0001 万亿几小时
100,00010^15几天到几月

而梯度下降每步只算 $O(nd)$（n 样本 × d 特征）。$d = 10000$ 也不过百万级别的操作，比 $10^{12}$ 小百万倍。

结论：

  • $d$ 小（几十～几百）→ 正规方程一步到位，最优雅
  • $d$ 大（几千以上）→ 矩阵求逆太慢甚至装不下内存，必须梯度下降
  • 损失没解析解 → 只能梯度下降

⚙️ Callback：等以后学完线性代数（矩阵乘法、向量、特征值），再回来读这段就完全清楚了。现在保留这个直觉就够："矩阵求逆 = 解一组线性方程组的标准操作，规模大时很慢"。

正规方程梯度下降
方法一步到位，公式直接算从初值出发，迭代逼近
结果数学最优（精确）趋近最优（够多步后无限接近）
计算量$O(d^3)$（矩阵求逆，d 大就崩）每步 $O(nd)$（d 千万级也能跑）
适用损失函数有解析解时任何能算梯度的损失

关键事实：线性回归很特殊，是少数有解析解的模型。你以后会遇到的几乎所有模型——逻辑回归、神经网络、深度学习——都没有解析解，只能靠梯度下降。所以这一章学梯度下降不是"线性回归用不上"，而是为整个机器学习世界打基础。本章继续用线性回归当练手对象，是因为它有解析解，可以用来对照梯度下降的结果是否正确。

4.1 直觉：下山

损失 $L(w, b)$ 是关于参数的函数。把它想象成一座山的高度：

  • 横轴：$w$ 和 $b$
  • 纵轴：损失 $L$
  • 山的最低点就是"最优参数"

你被空投在山上某处，蒙着眼怎么走到山底？答案：每一步都沿当前位置最陡的下坡方向走一小步。

每个红点是梯度下降一次迭代后的位置。绿色短线 = 该点的切线，斜率 = 那一点的导数——切线越陡，说明梯度越大，下一步就跨得越远。越接近最低点切线越平，步子也跟着变小，所以梯度下降会"自动减速"，平稳收敛到底部。

4.2 数学：梯度 = 最陡方向

梯度就是各方向偏导组成的向量：

$$\nabla L = \begin{pmatrix} \partial L / \partial w \\ \partial L / \partial b \end{pmatrix}$$

它指向函数上升最快的方向。所以下山就沿 $-\nabla L$ 走。

对线性回归，梯度具体是：

$$\frac{\partial L}{\partial w} = -\frac{2}{n}\sum x_i(y_i - \hat{y}_i)$$

$$\frac{\partial L}{\partial b} = -\frac{2}{n}\sum (y_i - \hat{y}_i)$$

4.3 更新规则

每次迭代：

$$w \leftarrow w - \eta \cdot \frac{\partial L}{\partial w}, \quad b \leftarrow b - \eta \cdot \frac{\partial L}{\partial b}$$

其中 $\eta$（读 eta）是学习率——每一步走多大。

到目前为止我们都在数据图上看训练：横纵轴是面积、价格，一条直线在动。但还有另一个完全不同的视角——把 $w, b$ 本身当成坐标轴，画"不同 $(w, b)$ 下损失有多大"。这一节解释这个新视角。

损失 $L(w, b)$ 在参数视角下是个二维曲面（碗形）。从正上方俯瞰，把"高度（损失）相同"的点连起来 = 等高线，像地形图：

⚠️ 这张图用了一个 9 个点的小型合成数据（不是房价数据）。原因：房价数据 $x$ 范围 50–140 太大，使 $w$ 和 $b$ 的曲率差几万倍，等高线变成极扁的椭圆几乎看不出"碗"的形状。换成数值平衡的合成数据后，等高线漂亮的椭圆形清晰可见。详见下方"为什么真实数据等高线扁"。

怎么读这张图（逐项对照）

  • 横轴 $w$、纵轴 $b$：当前考察哪一组参数
  • 背景颜色：浅 = 损失低、深 = 损失高（坐标点对应的 $L(w,b)$ 数值）
  • 红色细线 = 等高线，同一条线上所有点的损失相等。一圈套一圈的椭圆 = 损失"碗"从上往下被切成的等高横截面
  • 橙色折线 = 梯度下降的实际轨迹（每段就是 4.4 节点一次"走一步"）
  • 棕色点 = 起点（初始 $w, b$）
  • 绿色点 = 最优 $(w^*, b^*)$（碗的最低点）
观察两件事：
① 轨迹"垂直穿过"等高线。因为梯度方向 = 上升最快方向，几何上等价于垂直于等高线——顺着等高线走高度不变；垂直方向才让高度变化最快。所以梯度下降一定"垂直穿过等高线、朝低处走"。
② 起初步子大、靠近最优后步子变小。因为等高线密集 = 坡陡 = 梯度大 = 一步跨得远；靠近碗底等高线变疏 = 坡缓 = 步子自动变小。这就是 4.1 节"自动减速"的几何呈现。
为什么真实数据的等高线常常是扁椭圆？（实战陷阱）

这是优化里一个真实问题，叫"条件数"（ill-conditioning）。看看房价数据的实际曲率：

  • $w$ 方向的曲率 $\propto \sum x_i^2 / n \approx 9850$
  • $b$ 方向的曲率 $\propto 1$
  • 两个方向差近一万倍 → 等高线变成极扁的椭圆（一个方向 100 个单位，另一个方向 1 个单位才同等高度）

梯度下降在这种情况下会"之字形震荡"——快速跨过狭窄方向，缓慢爬行长方向，整体收敛慢。这就是为什么 4.4 节默认 lr 必须很小（0.00005）：太大了在 $w$ 方向就发散。

实战解决办法：特征标准化。每个特征都减均值除标准差，让所有特征都在相近的数值范围（比如均值 0、标准差 1）。标准化之后等高线接近圆，梯度下降收敛快得多。这是真实 ML 项目的必备预处理步骤。

真实场景小心：训练损失=0 通常是过拟合的信号！模型把训练数据"完美记住"了——意味着它把训练里的噪声也学了进去，对没见过的新数据反而预测得很差。
健康的训练曲线应该降到一个合理低点然后稳住，而不是无限往 0 走。后面学到正则化和验证集时，你会看到怎么主动避免这个陷阱。

⑤ 超参数

参数是模型学出来的（$w, b$）；超参数是你训练之前定下的（学习率、batch size、迭代次数等）。超参数选不好，模型再聪明也学不好。

5.1 学习率 η

梯度下降里每一步走多远。这是最重要的超参数。

学习率现象损失曲线
太小每步走得太短，要跑很久才到底缓慢下降
合适稳步收敛平滑下降到平台
偏大会震荡，但最终能下去抖动着下降
太大每步跨过最低点，越走越高发散，损失爆炸
在上一节的演示里把 η 拉到 0.003 试试，能直观看到"震荡"或"爆炸"。

5.2 Batch Size（批量大小）

每次更新参数用多少个样本计算梯度。

类型每次用多少特点
批量梯度下降 (BGD)全部 n 个准确但慢、内存开销大
随机梯度下降 (SGD)1 个快、有噪声、容易跳出局部极小
小批量 (Mini-batch)32 / 64 / 128 个实际最常用，平衡速度和稳定

具体数字感受：1000 样本 × 20 epoch 下，参数被更新多少次？

策略每次用多少样本每 epoch 更新次数20 epoch 总更新次数
BGD（全量）1000120
SGD（单样本）1100020,000
Mini-batch（batch=100）10010200

同样训练 20 个 epoch，SGD 更新参数的次数是 BGD 的 1000 倍！所以 SGD 走得快得多，但每一步只看一个样本，方向更"抖"。Mini-batch 卡在中间，工业标配。

SGD 的"随机性"是特性不是缺陷：梯度有噪声反而能跳出狭窄的局部极小，对复杂非凸损失（如神经网络）特别重要。线性回归的损失是凸的，SGD/BGD 都能收敛，但 mini-batch 是工业标准。
"一点噪声可能是好事"是个反直觉但非常重要的实战观点。

5.3 Epoch（轮次）

整个数据集被完整过一遍叫 1 个 epoch。训练通常要好几个 epoch。

  • 太少：欠拟合（还没学好就停了）
  • 太多：过拟合（开始记住训练集里的噪声）
  • 怎么定：看验证集损失什么时候不再下降（"早停"）

5.4 实战调参建议

  1. 先用合理默认值：η = 0.001~0.01，batch size = 32 或 64
  2. 如果损失爆炸 → 调小 η 一个量级
  3. 如果损失下降太慢 → 调大 η 一个量级
  4. 实际工程里常用学习率衰减：前期大、后期小

⑦ 编程练习

7.1 numpy 手写版（教学版）

10 套房子的数据：面积（m²）→ 价格（万元）。

import numpy as np

# 数据：[面积, 价格]
data = np.array([
    [50, 180], [60, 210], [70, 250], [80, 290], [90, 320],
    [100, 360], [110, 380], [120, 430], [130, 470], [140, 510],
])
X = data[:, 0]   # 面积
y = data[:, 1]   # 价格
n = len(X)

# 初始化参数
w, b = 0.0, 0.0
lr = 0.0001         # 学习率
epochs = 5000       # 迭代次数

# 训练循环
for epoch in range(epochs):
    # 1) 前向：算预测值
    y_hat = w * X + b

    # 2) 算损失（MSE）
    loss = np.mean((y - y_hat) ** 2)

    # 3) 算梯度（对 w 和 b 的偏导）
    dw = -2 * np.mean(X * (y - y_hat))
    db = -2 * np.mean(y - y_hat)

    # 4) 更新参数
    w -= lr * dw
    b -= lr * db

    if epoch % 1000 == 0:
        print(f"epoch={epoch:5d}  loss={loss:8.2f}  w={w:.4f}  b={b:.4f}")

print(f"\n最终: w={w:.4f}, b={b:.4f}, loss={loss:.2f}")
print(f"预测一套 95 m² 的房子: {w*95+b:.1f} 万元")
预期输出（点开看）

epoch= 1000  loss=   33.35  w=3.5843  b=-0.0567
epoch= 2000  loss=   33.27  w=3.5852  b=-0.1464
epoch= 3000  loss=   33.19  w=3.5860  b=-0.2345
epoch= 4000  loss=   33.12  w=3.5869  b=-0.3212

最终: w=3.5877, b=-0.4064, loss=33.04

7.2 sklearn 工程版（实战版）

import numpy as np
from sklearn.linear_model import LinearRegression

data = np.array([
    [50, 180], [60, 210], [70, 250], [80, 290], [90, 320],
    [100, 360], [110, 380], [120, 430], [130, 470], [140, 510],
])
X = data[:, 0:1]   # 注意：sklearn 要求 X 是 2D，所以用 0:1 而非 0
y = data[:, 1]

model = LinearRegression()
model.fit(X, y)

print(f"w = {model.coef_[0]:.4f}")
print(f"b = {model.intercept_:.4f}")
print(f"R² = {model.score(X, y):.4f}")
print(f"预测 95 m²: {model.predict([[95]])[0]:.1f} 万元")

b = -5.4545
R² = 0.9967

⑧ 测试你的知识

每题选一个答案，立即看到对错和解释。

Q1. 线性回归里的"线性"指的是？

"线性"指的是对参数线性。所以你完全可以构造 $x^2$ 作为新特征喂进去（叫多项式回归），模型仍然是"线性"的。

Q2. 为什么用平方损失而不用绝对值损失？

d 反了：MAE 对离群点更鲁棒，MSE 更敏感。c 才是平方损失被选中的真实原因。

Q3. 梯度下降里学习率太大会怎样？

学习率过大时，更新一步就跨过山谷到对面山坡，下次又跨回来 —— 来回震荡甚至越走越远。a 是"学习率太小"的现象。

Q4. 我们已经有正规方程能直接解析求解，为什么还要梯度下降？

正规方程是 $(X^\top X)^{-1} X^\top y$，d 维矩阵求逆是 $O(d^3)$。当 d 上万时根本算不动。梯度下降每次只算梯度（$O(nd)$），可扩展性远好于解析解。

Q5. 模型对一套房子预测 380 万，真实成交 420 万。这一条样本的平方损失贡献是多少？

$(420 - 380)^2 = 40^2 = 1600$。注意是平方，不是直接差值。

Q6. 训练时损失先降后升，可能是什么原因？

这是经典的"震荡发散"信号。解决：调小学习率（往往一个量级足够）。

Q7. 关于 MSE 和 RMSE，哪句正确？

RMSE = √MSE。因为开根号是单调变换，最小化它们得到的 $(w^*, b^*)$ 一样。报告 RMSE 因为单位匹配 y，"平均偏差 30 万"比"平均偏差 900 万²"好理解。

⑨ 后续步骤

立即能学的扩展

  • 多项式回归：把 $x^2, x^3$ 当成新特征喂进去，模型变成曲线，但训练算法不变
  • 正则化（L1/L2）：损失里加上 $\lambda \|w\|^2$，防止过拟合，叫 Ridge 回归；用 $\|w\|_1$ 叫 Lasso，能做特征选择
  • 逻辑回归：把线性输出过一个 sigmoid，变成概率，用来分类。底子和这里完全一样

真实场景里你还会遇到

  • 特征工程：原始数据通常需要标准化（$x \to (x - \mu)/\sigma$）、分箱、独热编码等才能喂进模型
  • 训练/验证/测试集：不能用同一份数据训练和评估，否则会高估模型能力
  • 解释性：每个 $w_i$ 的大小代表特征重要性，但要注意特征量纲影响 → 所以才需要标准化

推荐路径

  1. 把第 ⑦ 节的代码自己跑一遍，改 epochs 和学习率玩一下
  2. 找个真实数据集（kaggle 上的房价数据集 / sklearn 自带 California housing）跑一遍
  3. 学逻辑回归（结构相同，加个 sigmoid 而已）
  4. 学正则化和交叉验证
  5. 这时候你已经准备好学神经网络了 —— 它就是把线性层堆起来加非线性
