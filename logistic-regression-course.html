逻辑回归：从场景到代码

把"线性回归 → 概率 → 二分类"这条路一遍走通。承接前一篇线性回归。

🗺️ 这门课怎么读 —— 一句话串起每一节

核心问题：怎么从一堆 (学习时数, 是否通过考试) 数据里，学到一个能输出"通过概率"的模型？

这条路本质上是把线性回归改造一遍：模型外面套层 sigmoid 把输出压到 $[0,1]$ → MSE 在 sigmoid 上崩了，换成对数损失 → 数据线性可分时 $|w|$ 爆炸，加正则化兜底 → 最后落到代码。每一节都在解决上一节遗留的问题。

节学什么为什么放在这
⓪ 数学预热 条件概率 $P(y \mid x)$、独立事件连乘、$\ln$ 的两条性质 第 ③ 节推对数损失要用到 MLE，而 MLE 整条链就是"条件概率 → 连乘 → 取 log 变连加"。先把这些零件摆出来，到时候推导才不会卡在符号上
① 简介 分类 vs 回归的差别；直接用线性回归预测 0/1 为什么不行 用一个失败案例（线性回归画在 0/1 数据上）暴露出新需求：输出必须在 $[0,1]$ 之间，对极端值不能太敏感。下一节的 sigmoid 就是冲着这两条来的
② 计算概率 概率 → odds → log-odds 三步搭桥，反解出 sigmoid；1D / 2D 决策边界 ① 提出了"要 $[0,1]$ 输出"，这一节给出具体函数。关键 insight：sigmoid 不是临时拼的"压扁器"，是"log-odds 关于 $\mathbf{x}$ 线性"假设的代数副产品。模型有了，下一节才能给它配损失
③ 损失和正则化 为什么 MSE 在 sigmoid 上崩了；对数损失（从 MLE 推出）；线性可分时 $|w|$ 爆炸 → L1/L2 正则化 模型有了 → 必须给"好坏"一个数值标准。这节回答三个递进问题：(1) 沿用线性回归的 MSE 行不行（不行，非凸 + 梯度消失）；(2) 那用什么（对数损失，MLE 推出的）；(3) 配对数损失会出新问题（$|w|$ 飙到无穷），怎么治（正则化）
④ 编程练习 numpy 手写训练循环（含梯度下降 + L2）+ sklearn 调库；sklearn 的 C 参数 vs 我们的 $\lambda$ 前三节全是数学和直觉 → 这节落到代码。首次允许出现 np.exp、sklearn 这种代码术语。梯度公式 $(p - y) \cdot x$ 在 ③ 推出来了，这里直接拿来用
⑤ 测试 8 道选择题 检查盲点。比如"logistic 这个词到底指什么"（是 logit，不是逻辑判断）、"sklearn 的 C 越小是强还是弱正则化"（C = 1/λ，反着的）—— 都是容易混的地方
⑥ 后续 softmax 多分类、非线性边界、逻辑回归 → 神经网络的过渡 逻辑回归是单个神经元。把它堆起来就是神经网络的隐藏层，把 sigmoid 换成 softmax 就是多分类。这节给出下一步的路标

三条暗线（贯穿全课，不止一节）

  1. 每个设计都是 MLE 的副产品。sigmoid 来自"log-odds 线性"假设的代数推导（②），对数损失来自"伯努利分布 + 最大似然"的推导（③）。它们不是凑出来的，是同一个统计框架的两端 —— 跟线性回归 MSE 来自"高斯噪声 MLE"是完全平行的故事
  2. 和线性回归对照着学。模型上多套一层 sigmoid、损失从 MSE 换成对数损失、但梯度公式 $(p - y) \cdot x$ 和线性回归 $(\hat y - y) \cdot x$ 长得一模一样。会刻意在每一节标出"这里和线性回归的区别 / 相同"
  3. "为什么不行"驱动下一步。① 用线性回归预测 0/1 失败 → 推出 sigmoid；③ 沿用 MSE 失败 → 推出对数损失；③ 对数损失在线性可分数据上失败 → 推出正则化。每一节都在补上一节漏的洞

⓪ 数学预热

这节集中梳理后面 ③ 节"对数损失推导"会用到的几个概率与对数概念。已经熟悉的可以跳过；推导时遇到陌生符号再回来查。

0.1 条件概率 $P(y \mid x)$

竖线"$\mid$"念作 "在……条件下"。$P(y \mid x)$ 表示"已知输入是 $x$ 的前提下，标签为 $y$ 的概率"。

例子（学生考试场景）：

  • $P(y = 1 \mid x = 4)$ = "学习了 4 小时的学生通过考试的概率"
  • $P(y = 0 \mid x = 4)$ = "学习了 4 小时的学生没通过的概率"

因为只有两种结果，这两个加起来 = 1。逻辑回归模型直接给出第一个：

$$P(y = 1 \mid x) = p = \sigma(wx + b), \qquad P(y = 0 \mid x) = 1 - p$$

注意 $p$ 不是常数 —— 它依赖输入 $x$，也依赖参数 $(w, b)$。换不同的参数，每个 $x$ 对应的 $p$ 就变了。

0.2 独立事件的总概率 = 各自概率相乘

高中概率知识：若干相互独立的事件"全都发生"的概率 = 各自概率相乘。

例子：抛 3 次公平硬币全是正面，概率 = $0.5 \times 0.5 \times 0.5 = 0.125$。

机器学习里我们假设训练样本是独立抽出来的（标准假设）。所以"全部 $n$ 个观测都按真实标签出现"的总概率 = 每条样本各自概率的连乘：

$$P(\text{全部数据}) = P(y_1 \mid x_1) \cdot P(y_2 \mid x_2) \cdots P(y_n \mid x_n) = \prod_{i=1}^{n} P(y_i \mid x_i)$$

大写希腊字母 $\prod$（读 product）就是"连乘"号，跟 $\sum$（求和）是一对。

0.3 对数的两个核心性质

记号 $\ln$（自然对数）= 以 $e \approx 2.718$ 为底的对数。后面推导用到对数的两条性质：

性质 1（积变和）：

$$\ln(a \cdot b) = \ln a + \ln b, \qquad \ln(a^k) = k \ln a$$

对一长串连乘取 $\ln$ 就变成连加：$\ln \prod_i x_i = \sum_i \ln x_i$。这就是为什么似然取 log —— 把难算的连乘转成好算的连加。

性质 2（单调递增）：$a > b > 0 \implies \ln a > \ln b$。

所以"让 $f(x)$ 最大的 $x$" 和 "让 $\ln f(x)$ 最大的 $x$" 是同一个 $x$。换句话说，取 log 不改变最优解，只改变中间运算。

这两条性质合起来让 MLE 的推导可行：连乘对求导太麻烦、对计算机也容易下溢（很多小数相乘 → 接近 0 → 算不准），而取 $\ln$ 之后变成连加，求导轻松、数值稳定，且最优解不变。

① 简介：从"预测多少"到"预测是/否"

真实场景：你是教务系统的开发者。已知 15 个学生的考前学习时数和是否通过考试， 现在新来一个学生说"我学了 4.2 小时"，他通过的概率有多大？

1.2 看一下数据

每个点的 y 轴只有两个值：通过（1）或不通过（0）。

直觉上：学得越多，通过概率越高。但不是绝对的——有些学得多的也没过，有些学得少的反而过了（运气、基础、题目难度等噪声）。

1.3 第一个错误尝试：直接用线性回归

"既然 y 只是 0 或 1，那线性回归直接学 $\hat{y} = wx + b$ 不就完了？"我们试试看：

这条线有三个明显的问题：

  1. 输出会超出 [0, 1]。学了 0.5 小时的人，模型说"通过概率 = -0.15"？通过概率 = 1.3？没意义。
  2. 对离群点过度敏感。一个"学了 10 小时但没过"的点会大幅扭曲直线，因为平方损失对偏差按距离平方惩罚，0/1 标签的偏差太大。
  3. 预测和真实标签的语义不一致。直线给的是连续值，但我们想要的是"概率"或"类别判断"。
我们需要一个模型，输出一定在 [0, 1] 之间（可以解释成概率），并且对极端值不敏感（学了 100 小时也只能更"几乎确定通过"，不能无限大）。这就是逻辑回归的核心动机。

1.4 解决方案预览

逻辑回归 = 线性回归 + 一个把输出压到 [0,1] 的"S 形函数"。下一节展开它怎么来的。

蓝色 S 曲线就是逻辑回归的输出 —— 永远在 0 和 1 之间。

② 计算概率：sigmoid 是怎么来的

我们想要的目标：输入 $\mathbf{x}$ → 输出一个 0~1 的概率 $p$。但线性组合 $\mathbf{w}^\top \mathbf{x} + b$ 的取值范围是 $(-\infty, +\infty)$。怎么搭桥？

2.1 三步搭桥：概率 → 几率 → 对数几率

Step 1：概率 → 几率（odds）

$$\text{odds} = \frac{p}{1-p}$$

赌博和保险里常用。"通过概率 0.8"等价于"几率 4 比 1"。

p（概率）odds（几率）
0.10.11
0.51.0
0.99.0
0.9999.0

取值范围：$[0, +\infty)$。比概率宽了一边，但下界还是 0。

Step 2：几率 → 对数几率（log-odds，又叫 logit）

$$\text{logit}(p) = \ln \frac{p}{1-p}$$

对几率再取一个对数，取值范围一下子扩到 $(-\infty, +\infty)$ —— 正好和线性组合的范围对上了！

⚠️ 注意：这条曲线不是直线 —— logit 函数本身关于 $p$ 是非线性的（S 形拉直，在 $p=0, 1$ 处往 $\pm\infty$ 爆炸）。下面 Step 3 说"log-odds 线性"不是说这条曲线变直了，指的是另一回事：$\text{logit}(p)$ 关于特征 $\mathbf{x}$ 才是直线关系。两个"线性/非线性"不是同一件事，下面对比表里再展开。

Step 3：关键假设 —— log-odds 是线性的

$$\ln \frac{p}{1-p} = \mathbf{w}^\top \mathbf{x} + b = z$$

这才是"线性"的真正含义 —— 横轴特征 $x$、纵轴 $\text{logit}(p)$，是一条直线（不是 Step 2 那条 S 形曲线）：

举例：假设 $w = 0.5,\ b = -3$，即 $\text{logit}(p) = 0.5x - 3$。三个红点演示"特征值 → logit → 概率"的完整链：
• $x = 0$（没学习）→ logit $= -3$ → $p \approx 0.047$（几乎不通过）
• $x = 6$（学 6 小时）→ logit $= 0$ → $p = 0.5$（决策边界，最不确定）
• $x = 12$（学 12 小时）→ logit $= 3$ → $p \approx 0.953$（几乎一定通过）
关键观察：logit 关于 $x$ 是直线（每多学 1 小时 logit 增加 0.5，斜率恒定）。但反解回概率 $p$，就成了你在 1.4 节看到的 sigmoid 形状 —— 同一条直线、两种视角。

这是逻辑回归的本质假设。线性回归说"$y$ 是 $\mathbf{x}$ 的线性函数"，逻辑回归说"$\text{logit}(p)$ 是 $\mathbf{x}$ 的线性函数"。线性模型族的"线性"指的是参数 $\mathbf{w}, b$ 出现的方式，不是说 $p$ 和 $\mathbf{x}$ 是线性关系，更不是说 logit 函数本身（上图那条曲线）是直的。

真正的数学顺序其实跟教学反着讲。本节按"想要 0-1 概率 → 线性范围对不上 → 造 sigmoid"教，但真实数学故事是从另一头来：
教学顺序（本节）数学顺序（真实故事）
起点想要 $p \in [0,1]$假设 $\text{logit}(p) = \mathbf{w}^\top \mathbf{x} + b$ 是线性
看到的"矛盾"$\mathbf{w}^\top \mathbf{x} + b$ 范围 $(-\infty, +\infty)$ 跟 $[0,1]$ 对不上没矛盾 —— 左右两边都是 $(-\infty, +\infty)$，天然吻合
动作"造一个 sigmoid 把线性压扁到 $[0,1]$"反解 $p$ → $p = \frac{1}{1+e^{-(\mathbf{w}^\top\mathbf{x}+b)}}$（自动是 sigmoid 形式）
结果sigmoid + 线性 = 逻辑回归同一个模型，只是导出路径不同
关键 insight：sigmoid 不是临时拼凑的"压扁器"，而是"log-odds 关于 $\mathbf{x}$ 线性" 这条建模假设的代数副产品。教学倒着讲只是因为读者还没听说过 log-odds，从概率视角入手更直观。

这条线性假设直观吗？翻译成人话："每多 1 单位特征，赢面（几率 odds）乘以一个固定倍数"。乘法效应取对数自动变成加法（也就是线性）。所以听起来抽象的"log-odds 线性"，本质上是个跟"复利"、"传染率"一个套路的自然假设。

展开一层：具体数字 + 为什么"乘法效应"是自然假设（可跳过）

具体数字感受一下。设基线几率（不学习时）= 1:20 = 0.05，假设每多学 1 小时几率 ×1.65：

  • $x = 0$：odds $= 0.05$ → $p \approx 0.048$（几乎不通过）
  • $x = 6$：odds $= 0.05 \times 1.65^6 \approx 1.01$ → $p = 0.50$（决策边界）
  • $x = 12$：odds $= 0.05 \times 1.65^{12} \approx 20.4$ → $p \approx 0.953$（几乎一定通过）

几率本身是指数曲线（$0.05 \to 1 \to 20$，一路飞涨），但取 $\ln$ 后立刻变直线：$\ln(\text{odds}) = \ln(0.05) + x \cdot \ln(1.65) \approx -3 + 0.5x$。这正是上面 SVG 里那条蓝线。

为什么"乘法效应"是自然假设？三层理由：

  1. 边界问题：概率被卡在 $[0,1]$。"每小时加固定概率"在边界附近会爆掉（如 $p = 0.05 + 10 \times 0.1 = 1.05 > 1$）。但 odds 没上界（$[0, +\infty)$），乘法自动绕过这个问题。
  2. 相对影响比绝对影响更稳定：人脑对"翻倍 / 减半"敏感（复利、价格折扣都是这套），对"加 0.1"不敏感（"5% 提升到 15%"和"50% 提升到 60%"虽然都加 10%，但难度天差地别）。乘法效应捕捉的是相对变化，更符合"影响大小"的直觉。
  3. 贝叶斯定理本来就这样：log 空间下，"后验 log-odds = 先验 log-odds + 每条证据贡献的 log 似然比"，是天然的加法结构。如果每个特征贡献固定 log 似然比 → 总 log-odds 关于特征线性。

⚙️ 若想再深一层：理由 3 在 LDA 假设下能严格证明（两类特征都是协方差相同的高斯分布时，logit 关于 $\mathbf{x}$ 精确是线性）。但这些理论锚点不影响使用 —— 知道"乘法效应"这层直觉就够了。

2.2 反解：从 log-odds 回到概率

把 $\ln \dfrac{p}{1-p} = z$ 解出 $p$：

$$\frac{p}{1-p} = e^z \implies p = (1-p)e^z \implies p(1 + e^z) = e^z$$

$$\boxed{p = \frac{e^z}{1 + e^z} = \frac{1}{1 + e^{-z}} = \sigma(z)}$$

这就是 sigmoid 函数。它的图像：

−∞0笃定不发生
−30.047几乎不会
00.5决策边界（最不确定）
+30.953很可能
+∞1笃定发生

2.3 完整模型

$$p = \sigma(\mathbf{w}^\top \mathbf{x} + b) = \frac{1}{1 + e^{-(\mathbf{w}^\top \mathbf{x} + b)}}$$

预测时：

  • 如果 $p > 0.5$（即 $z > 0$）→ 类别 1（通过）
  • 如果 $p < 0.5$（即 $z < 0$）→ 类别 0（不通过）
  • $p = 0.5$ 的位置叫决策边界

2.5 二维情况：决策边界是一条直线

先理清"决策边界"在 1D vs 2D 的含义

特征数决策边界是什么
1D：仅学习时长 $x$$x$ 轴上一个点（$p = 0.5$ 处的临界 $x$ 值，前面例子是 $x = 6$）
2D：学习 $x_1$ + 睡眠 $x_2$$(x_1, x_2)$ 平面上一组点 —— 所有让 $p = 0.5$ 的组合连起来

2D 的决策边界长什么样

加一个特征"睡眠时数"后，模型变成 $z = w_1 x_1 + w_2 x_2 + b$。下图用的参数是 $w_1 = 0.8,\ w_2 = 0.6,\ b = -7$，所以：

$$p = 0.5 \;\Leftrightarrow\; z = 0 \;\Leftrightarrow\; 0.8 x_1 + 0.6 x_2 - 7 = 0$$

代几组数（都让 $p = 0.5$）：

$x_1$ 学习$x_2$ 睡眠验证 $z$含义
2 h9 h$1.6 + 5.4 - 7 = 0$学得少但睡得多，勉强通过
5 h5 h$4.0 + 3.0 - 7 = 0$各一半，勉强通过
6.5 h3 h$5.2 + 1.8 - 7 = 0$学得多但睡得少，勉强通过

这三个点都"勉强能通过" —— 各种"学习与睡眠"的折中。它们和无数个类似的点连起来，就是下图那条紫色直线。

为什么这条线一定是直的

$0.8 x_1 + 0.6 x_2 - 7 = 0$ 是 $(x_1, x_2)$ 的一次方程，几何上必然是直线（中学学的 $ax + by + c = 0$）。这跟线性回归 4.6 节"等高线必然是椭圆"是同种代数推理：

  • 线性回归的等高线方程是二次方程 → 椭圆
  • 逻辑回归的决策边界方程是一次方程 → 直线

所以"决策边界是直线"不是建模选择，是 sigmoid + 线性 这套组合的代数必然。

背景颜色 = sigmoid 输出的 $p$（蓝色 = 高概率通过、红色 = 高概率不通过）。紫色直线 = $p = 0.5$ 处。关键观察：沿直线方向走 $p$ 不变（线上所有点都是 0.5）；垂直于直线方向走 → 从远端红到远端蓝平滑过渡，这就是 sigmoid 的 S 形"展开"在 2D 里的样子。

维度升级的规律

特征数决策边界几何方程
1一个点$w_1 x_1 + b = 0$
2一条直线$w_1 x_1 + w_2 x_2 + b = 0$
3一个平面$w_1 x_1 + w_2 x_2 + w_3 x_3 + b = 0$
$n$超平面（$n-1$ 维）$\mathbf{w}^\top \mathbf{x} + b = 0$

每多一个特征，决策边界的维度 +1（点 → 线 → 面 → 超平面），但永远"线性"。

逻辑回归的决策边界永远是直线 / 平面 / 超平面。要想拟合弯曲的边界，得加特征工程（比如加 $x_1^2, x_1 x_2$，把弯曲塞进特征里），或用神经网络等更复杂模型。

③ 损失和正则化

3.0 写损失之前：连续概率 vs 离散标签

正式定义损失之前，先把一个核心差别讲清楚 —— 这是逻辑回归和线性回归在概念上最根本的分歧。

两边类型不一样：

量类型取值例子
真实标签 $y$离散只能是 $0$ 或 $1$$1$（学生通过了考试）
模型预测 $p$连续$(0, 1)$ 间任意实数$0.73$（模型猜"73% 像通过"）

线性回归里没这个问题 —— $\hat y$ 和 $y$ 都是连续值（都是房价），直接做差就有"距离"的物理意义。而在逻辑回归里，连续的 $p$ 和离散的 $y$ 不在同一把尺子上，没法直接说"差多远"。我们需要一个翻译器，把它们对齐到同一个标量（损失值）。损失函数的本职工作就是当这个翻译器。

为什么不直接把 $p$ 也压成 0 或 1？

那样确实类型对齐了，但 $p$ 一旦被压成 0/1 就求不了导（阶跃函数到处导数为 0 或不存在）→ 梯度下降没法 work。所以策略反过来：保留 $p$ 的连续性，让损失函数承担"拉近连续 $p$ 和离散 $y$"的责任。

两个阶段，$p$ 扮演不同角色

阶段$p$ 怎么用是否阈值化
训练（算 loss、求梯度）保持连续❌ 必须保留 $(0,1)$ 实数
预测（输出最终判断）卡 0.5 阈值（$>0.5$ 判 1）✅ 此时才压成 0/1

训练时 $p$ 必须连续，否则梯度不存在；预测时才阈值化。所以训练目标其实是 "让 $p$ 在 $y=1$ 的样本上往 1 靠，在 $y=0$ 的样本上往 0 靠" —— 损失函数就是衡量"靠得有多近"的那把尺子。

下面 3.1 和 3.2 会看到两个候选翻译器（MSE 和对数损失）。两个都"能用"，区别是哪一个翻译得让 GD 学得动。

3.1 为什么不能用 MSE？

有了模型 $p = \sigma(z)$，自然的想法是延续线性回归的平方损失：$L = (p - y)^2$。但有两个数学灾难：

灾难 1：损失面不是碗形 → 有大片平坦区

线性回归的 MSE 关于 $(w, b)$ 是凸函数（碗形）。但 $\sigma$ 是非线性的，外面再套一个平方后，损失面被"压扁"成两端平坦、中间陡的形状 —— 不是碗。下图蓝线是单样本 $(y=1, x=1)$ 下 MSE 关于 $w$ 的曲线：

左半边几乎是水平线 —— 这片"平台"上梯度接近 0，参数更新几乎为零。梯度下降一旦初始化在这里就动弹不得。红色虚线是对数损失（下一节会讲），左侧陡峭、梯度大，能把参数推回正确方向。

灾难 2：梯度消失 → 自信地错时反而学不动

上面"平台"现象的根源，从梯度公式能看出来。对 MSE+sigmoid 求 $w$ 的偏导（链式法则）：

$$\frac{\partial L}{\partial w} = \underbrace{(p - y)}_{\text{误差}} \cdot \underbrace{\sigma'(z)}_{= p(1-p)} \cdot x$$

这个梯度是怎么求出来的？（链式法则三步）

把 $L$ 一层层拆开：

  • $L = (p-y)^2$ —— 损失是 $p$ 的函数
  • $p = \sigma(z)$ —— $p$ 是 $z$ 的函数
  • $z = wx + b$ —— $z$ 是 $w$ 的函数

$L$ 依赖 $p$，$p$ 依赖 $z$，$z$ 依赖 $w$。链式法则把每一步的导数串起来：

$$\frac{\partial L}{\partial w} = \underbrace{\frac{\partial L}{\partial p}}_{①} \cdot \underbrace{\frac{\partial p}{\partial z}}_{②} \cdot \underbrace{\frac{\partial z}{\partial w}}_{③}$$

  • ①：$L = (p-y)^2$ 对 $p$ 求导 → $2(p-y)$
  • ②：$p = \sigma(z)$ 对 $z$ 求导 → $\sigma'(z) = p(1-p)$（上面刚证过）
  • ③：$z = wx + b$ 对 $w$ 求导 → $x$

三个乘起来：

$$\frac{\partial L}{\partial w} = 2(p-y) \cdot p(1-p) \cdot x$$

那 2 去哪了？ 正文公式里把常数 2 省掉了。理由是 2 可以并入学习率（把 lr 看成 "2×lr" 就行），不改变梯度方向，也不影响"梯度是否接近 0"的判断，所以教科书习惯简写成 $(p-y) \cdot p(1-p) \cdot x$。如果想让公式严格干净，常见做法是把损失定义为 $L = \frac{1}{2}(p-y)^2$ —— 这个 $\frac{1}{2}$ 就是为了求导后把 2 抵消。

第一个因子 $(p-y)$ 是误差，错得越离谱越大，这没问题。第二个因子 $\sigma'(z) = p(1-p)$ 是 sigmoid 的导数 —— 注意它恰好可以写成 $p$ 和 $1-p$ 的乘积。这个因子才是问题所在。

顺手看一下 $\sigma'(z) = p(1-p)$ 是怎么来的（可选）

$\sigma(z) = \dfrac{1}{1+e^{-z}}$，对 $z$ 求导：

$$\sigma'(z) = \frac{e^{-z}}{(1+e^{-z})^2} = \frac{1}{1+e^{-z}} \cdot \frac{e^{-z}}{1+e^{-z}} = \sigma(z)\cdot(1-\sigma(z)) = p(1-p)$$

$p(1-p)$ 长什么样：它是一条倒 U 形抛物线 —— 在 $p=0.5$ 处取得最大值 0.25，在 $p=0$ 或 $p=1$（也就是 sigmoid 两端饱和的地方）跌到 0：

具体数值：

pp(1-p)梯度规模
0.50.25正常
0.90.09缩水
0.990.0099已经很小
0.001（自信预测错的极端）0.000999梯度几乎为 0

为什么"小"会出事：梯度下降每一步的更新量是 $\Delta w = -\text{lr} \cdot \dfrac{\partial L}{\partial w}$。梯度 ≈ 0 → 更新 ≈ 0 → 参数几乎不动 → 学不动。

最糟的情况：模型自信地预测错（真实 $y=1$ 但 $p=0.001$）。误差因子 $(p-y) = -0.999$ 很大，本该狠狠惩罚；但 $p(1-p) \approx 0.001$ 几乎为零，两者相乘梯度还是 ≈ 0。错得最离谱时反而最学不动 —— 这就是"梯度消失"。

3.2 对数损失（Log Loss / Cross-Entropy）

正确的损失函数：

$$L = -\big[\,y \ln p + (1-y) \ln(1-p)\,\big]$$

看着复杂，其实是个 if/else 开关：

  • 当 y = 1：损失 = $-\ln p$ —— 罚"你给 1 的概率太低"
  • 当 y = 0：损失 = $-\ln(1-p)$ —— 罚"你给 0 的概率太低"

它是怎么推出来的？最大似然估计 (MLE)

核心思想（一句话）：找一组 $(w, b)$，让模型对训练数据中真实标签的"信心"加起来最大 —— 也就是让"模型生成这批观测数据"的概率最高。这种思路叫最大似然估计（Maximum Likelihood Estimation，MLE）。对数损失不是凭空设计的"惩罚函数"，而是从 MLE 推出来的副产品。

预备知识在 ⓪ 数学预热（条件概率、独立相乘、对数性质）。下面推导每一步都会标出引用了哪条。

展开看完整推导（六步）

Step 1：写出单样本的条件概率（用 ⓪.1）

模型对每个样本输出概率 $p = \sigma(wx+b)$。所以：

$$P(y = 1 \mid x) = p, \qquad P(y = 0 \mid x) = 1 - p$$

Step 2：用 $y$ 当指数开关，把两条合成一条

上面要分两段写很啰嗦。利用 $y$ 只能取 0 或 1，可以用一个数学小把戏写成一行：

$$P(y \mid x) = p^y \cdot (1-p)^{1-y}$$

验证它和原来两条等价：

$y$ 代入化简对应原式
$y = 1$$p^1 \cdot (1-p)^0 = p \cdot 1 = p$$= P(y=1 \mid x)$ ✓
$y = 0$$p^0 \cdot (1-p)^1 = 1 \cdot (1-p) = 1-p$$= P(y=0 \mid x)$ ✓

关键观察：$y$ 和 $1-y$ 充当指数开关—— 哪一项"打开"取决于 $y$ 是 0 还是 1。这个式子叫伯努利分布的概率函数（名字不用记，懂这个开关结构就行）。好处是后面公式可以一行写完，不用分情况讨论。

Step 3：n 个样本独立 → 总概率 = 各自相乘（似然函数）（用 ⓪.2）

训练样本独立抽样，"全部 $n$ 个观测都按真实标签出现"的总概率：

$$\mathcal{L}(w, b) = \prod_{i=1}^{n} P(y_i \mid x_i) = \prod_{i=1}^{n} p_i^{y_i}(1-p_i)^{1-y_i}$$

这个 $\mathcal{L}$ 就叫似然函数（likelihood）。注意它是关于 $(w, b)$ 的函数 —— 数据 $x_i, y_i$ 是固定的训练集，$p_i = \sigma(wx_i + b)$ 随 $(w, b)$ 变。

MLE 的目标：找让 $\mathcal{L}$ 最大的那组 $(w, b)$。直觉上就是"哪组参数最能解释训练数据"。

Step 4：取对数，把连乘变连加（用 ⓪.3 性质 1）

连乘有两个问题：(a) 数值上很多小数相乘会下溢成 0；(b) 数学上乘法不好求导。
解法：取 $\ln$ 之后积变和，再用 $\ln(a^k) = k\ln a$ 把指数拉下来：

$$\ln \mathcal{L} = \sum_{i=1}^{n} \ln\big[p_i^{y_i}(1-p_i)^{1-y_i}\big] = \sum_{i=1}^{n}\big[y_i \ln p_i + (1-y_i)\ln(1-p_i)\big]$$

这一步没改变最优解：因为 $\ln$ 单调递增（⓪.3 性质 2），让 $\mathcal{L}$ 最大的 $(w, b)$ 同时也让 $\ln \mathcal{L}$ 最大。

Step 5：取负号，把"最大化"翻成"最小化"

我们整个机器学习的优化框架都是"最小化损失"。但 MLE 是"最大化似然"，方向反过来。加个负号就把它转成最小化：

$$L(w, b) = -\ln \mathcal{L} = -\sum_{i=1}^{n}\big[y_i \ln p_i + (1-y_i)\ln(1-p_i)\big]$$

这就叫负对数似然（Negative Log Likelihood, NLL），也就是对数损失。"最大化似然"和"最小化对数损失"是同一回事，只是符号反一下。

Step 6：单样本形式 = 课程主公式

把 $n=1$（看单条样本的损失贡献）写出来：

$$L = -\big[\,y \ln p + (1-y) \ln(1-p)\,\big]$$

这就是 3.2 一开始那个公式。整套推导走完了。

整条链关键观点：log 不是"为了惩罚大错"故意加的，而是为了把连乘转成连加自然出现的；$y$ 当指数开关也不是凑出来的写法，而是伯努利分布的标准形式。对数损失是 MLE 推导的副产品，不是设计目标。
跟线性回归 MSE 的"对应高斯噪声 MLE"是完全平行的故事：那边假设噪声是正态分布、MLE 推出平方损失；这边假设标签服从伯努利分布、MLE 推出对数损失。两个损失都有统计学根基。

对数损失的两个优良性质

性质 1：自信地错 → 损失爆炸到无穷

真值 y=1 时：

预测 pMSE 损失 (1-p)²对数损失 -ln p

性质 2：和 sigmoid 配，梯度异常干净

$$\frac{\partial L}{\partial w} = (p - y) \cdot x$$

看，没有 $p(1-p)$ 因子了！它和 sigmoid 求导带来的 $p(1-p)$ 在数学上正好抵消了。

回想线性回归的梯度是 $(\hat{y} - y) \cdot x$。逻辑回归的梯度长得一模一样，只是把 $\hat{y}$ 换成 $p$。这种统一性是"对数损失 + sigmoid"这一对刻意搭配的回报。

3.3 正则化：防止过拟合

逻辑回归的过拟合长什么样？

训练里如果数据正好"能被一条直线分开"（线性可分——一侧全 $y=1$、另一侧全 $y=0$，中间没有任何"骑墙"点），逻辑回归会做一件反直觉的事：训练永远不收敛，$|\mathbf{w}|$ 一路飙到无穷大。

为什么停不下来？

回想 3.2 节，每个点的损失是 $-\ln p$（当 $y=1$）或 $-\ln(1-p)$（当 $y=0$）。这个损失只在 $p$ 严格等于 1 或 0 时才为 0。但 sigmoid 永远取不到 0 或 1，只能无限接近：

$z = wx+b$$p = \sigma(z)$$-\ln p$（损失）

每把 $z$ 放大一倍，损失就再少一截。因为 $z = wx + b$，放大 $w$ 就是放大 $z$。只要数据真的线性可分，优化器每一步都会发现"$w$ 翻倍一下，损失还能再降"——于是 $|w|$ 一路飙升，收敛不到任何有限值。

$|w|$ 大了到底哪里坏？

先看一个几何现象，再看它带来的两个后果。

几何现象：sigmoid 退化成阶跃函数。 sigmoid 的"过渡区"（从 $p=0.1$ 升到 $p=0.9$ 那段）宽度大约是 $4.4/|w|$。$|w|=1$ 时过渡区有 4.4 个单位宽，$|w|=100$ 时只剩 0.044——决策边界从"缓坡"变成"一刀切"，边界两侧的点全被赋 $p \approx 1$ 或 $p \approx 0$，"中间地带"消失。

"中间地带"消失会带来两个直接后果：

  1. 过度自信 → 测试集翻车：训练点都被赋 $p \approx 1$ 或 $p \approx 0$，模型说"我 99.999% 确定"。但测试集点稍微挪一点（哪怕一点点噪声），就可能跨过那个无限陡的边界，预测从 1 翻到 0，错得彻底。边界越陡，模型对小扰动越敏感——这就是"中间地带消失"的直接物理表现。
  2. 概率失去意义：逻辑回归比硬分类器值钱就值钱在它输出概率——能表达"非常确定"还是"将将过线"。"中间地带"消失之后概率退化成 0/1，这个区分能力没了，模型变成一个比阶跃函数贵一点的硬分类器。
关于 $b$：严格说，线性可分场景里 $b$ 也跟着爆——上面那个 100000 步训练里 $w \approx 6.82, b \approx -30.58$，$b$ 反而长得更狠。但决定 sigmoid 陡度的是 $|w|$（过渡区宽度 $\approx 4.4/|w|$，跟 $b$ 无关），$b$ 只决定决策边界位置（$x = -b/w$）。所以"sigmoid 变阶跃"这个核心论点只跟 $|w|$ 有关，聚焦 $|w|$ 是抓主因；完整说应该是参数范数 $\|(\mathbf{w}, b)\|$ 一起膨胀。

互动：看 $|w|$ 怎么爆炸

数据强行线性可分：x ∈ {1,2,3,4} 全是 0，x ∈ {5,6,7,8} 全是 1，中间留空（没有任何骑墙点——只要中间有一个夹在两类之间的点，最优 $w$ 就有限了）。拖动训练步数看 sigmoid 怎么从"缓坡"变成"台阶"：

|w| = — 训练损失 = — 过渡区宽度 = —
拖到 10 万步：$|w| \approx 6.8$，sigmoid 的过渡区只剩约 0.64 个单位宽，曲线几乎垂直。但训练损失还是 $\approx 0.008$、永远到不了 0——因为 sigmoid 永远取不到 1。再训下去 $|w|$ 还会继续涨，损失还会更小一点点，但优化器永远没有"停下来"的理由。

解决：在损失里加"权重大小的惩罚"

思路：原来损失只关心"拟合数据多准"，现在再加一项关心"参数有多大"，两项相加做总损失。参数越大，总损失越大，梯度下降就会自己把 $|w|$ 拉回合理范围、不会爆。

"参数有多大"怎么量？两种最直观的办法：

  • 把所有 $w_j$ 平方加起来：$w_1^2 + w_2^2 + \dots$ → 叫 L2 正则化（也叫 Ridge）
  • 把所有 $w_j$ 取绝对值加起来：$|w_1| + |w_2| + \dots$ → 叫 L1 正则化（也叫 Lasso）

"L1 / L2" 里的 "L" 来自数学里的 $L_p$ 范数（一族衡量"向量大小"的方式）：$p=2$ 时 = 先平方再求和（再开方），就是几何里直线长度；$p=1$ 时 = 绝对值求和。本课不展开范数理论，记住 "L2 平方、L1 绝对值" 就够用。

L2 正则化（Ridge）的完整损失：

$$L_{\text{total}} = L_{\text{logloss}} + \lambda \sum_j w_j^2$$

$\lambda$（读 lambda）是正则化强度超参数：$\lambda = 0$ 退化成不加正则；$\lambda$ 越大越强烈压制 $w$。

L1 正则化（Lasso）的完整损失：

$$L_{\text{total}} = L_{\text{logloss}} + \lambda \sum_j |w_j|$$

跟 L2 只差一个绝对值（平方 → 绝对值），但行为大不一样：L1 会让一部分 $w_j$ 精确等于 0（下面解释为啥）。

展开：Ridge / Lasso 名字怎么来的、L2 为啥不带根号（可跳过，纯典故）

Ridge（岭回归）：来自线性回归求最优 $w$ 的"正规方程"——里面要算一个矩阵的逆。加 L2 正则相当于在那个矩阵的主对角线上"加一道山脊"（ridge），让矩阵更稳定（即使本来求不了逆的情况也能求）。涉及矩阵代数细节，不影响理解 L2 本身。

Lasso：Least Absolute Shrinkage and Selection Operator 的首字母缩写——同时"压缩"（让 $w$ 变小）和"选择"（精确选哪些 $w$ 归 0）。

L2 公式为啥不带根号：数学上 L2 范数本身是 $\sqrt{\sum w_j^2}$（向量长度），但放进损失里我们直接用 $\sum w_j^2$（去掉根号）。原因是去根号后对 $w$ 求导更干净（变成 $2w$，没烦人的根号链式），最优 $w$ 的方向跟带根号版本一致，只是数值缩放不同。

为什么 L1 能把权重精确归 0，L2 不能？

看正则项对 $w$ 的梯度（数据损失梯度暂时不管）：

正则项梯度$w$ 接近 0 时的拉力
L2: $\lambda w^2$$2\lambda w$越小越弱（$w \to 0$ 时拉力 $\to 0$）
L1: $\lambda |w|$$\lambda \cdot \text{sign}(w)$恒定 $\lambda$（永远拉同样狠）

L2 把 $w$ 推向 0 但"懒得"拉到 0（快到 0 时拉力消失），停在很小的非零值；L1 哪怕 $w$ 已经很小也照样恒定拉，数据损失的微弱推力顶不住，$w$ 直接被推过 0、停在 0 上。所以 L1 适合"很多特征但只有少数重要"的场景，L2 适合"所有特征都有点用"的场景。

等等，加了惩罚不会让损失"降不下去"吗？ 常见疑问。直觉错在把损失看成"单一目标"——其实总损失是两个对抗力的合：
  • $L_{\text{data}}$：想让 $|w|$ 变大去拟合数据——但 $|w|$ 越大它能再降的空间越小
  • $\lambda \|w\|^2$：想让 $|w|$ 变小——但 $|w|$ 越大它涨得越快（$w^2$ 是抛物线）
GD 收敛到一个平衡点 $w^*$：在那里两边梯度正好抵消（数据梯度 + 正则梯度 = 0）。这个 $w^*$ 是有限值——不像不加正则时 $|w| \to \infty$。

橡皮筋类比：把 $w$ 想成一颗珠子，一根橡皮筋被"数据"往外拉，一根被"正则"往 0 拉，两根都是越拉越紧。最终珠子停在两边拉力相等的位置——既不到 0（数据橡皮筋顶住），也不到无穷（正则橡皮筋顶住）。

关键认知：加正则后最终的训练损失会比"不加正则时的训练损失"大——但不加时那个"小损失"是过拟合的产物（训练集贴近 0、泛化烂）。加正则后这个"中等损失"对应的 $w$ 是有限的、泛化更好的 $w$，这正是要的。训练损失不再是唯一目标，泛化才是。

场景选谁原因

4.1 numpy 手写版（含正则化）

import numpy as np

# 学习时数 → 通过(1)/不通过(0)
X = np.array([1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0, 4.5, 5.0, 5.5, 6.0, 6.5, 7.0, 7.5, 8.0])
y = np.array([  0,   0,   0,   0,   0,   1,   0,   1,   1,   0,   1,   1,   1,   1,   1])
n = len(X)

# 初始化
w, b = 0.0, 0.0
lr = 0.1
epochs = 5000
lam = 0.01   # L2 正则化系数

def sigmoid(z):
    return 1.0 / (1.0 + np.exp(-z))

for epoch in range(epochs):
    # 1) 前向：算预测概率
    z = w * X + b
    p = sigmoid(z)

    # 2) 算损失（log loss + L2 正则项）
    eps = 1e-12   # 防止 log(0)
    loss = -np.mean(y * np.log(p + eps) + (1-y) * np.log(1-p + eps))
    loss += lam * (w ** 2)

    # 3) 梯度
    dw = np.mean((p - y) * X) + 2 * lam * w
    db = np.mean(p - y)

    # 4) 更新
    w -= lr * dw
    b -= lr * db

    if epoch % 1000 == 0:
        acc = np.mean((p > 0.5).astype(int) == y)
        print(f"epoch={epoch:5d}  loss={loss:.4f}  acc={acc:.2%}  w={w:.3f}  b={b:.3f}")

# 预测
print(f"\n最终: w={w:.3f}, b={b:.3f}")
print(f"学了 4.2 小时通过概率: {sigmoid(w*4.2+b):.2%}")
print(f"学了 6.5 小时通过概率: {sigmoid(w*6.5+b):.2%}")
print(f"决策边界（p=0.5）: x = {-b/w:.2f} 小时")

4.3 sklearn 的 C 参数 vs 我们的 λ

sklearn 用 C = 1/λ 作为参数，方向反过来的：

  • C 大（λ 小）→ 正则化弱 → 拟合更激进
  • C 小（λ 大）→ 正则化强 → 拟合更保守
这是初学者常见的坑。C=0.01 在 sklearn 里是强正则化，不是弱。

4.4 课后练习

  1. 把 lam 调成 0、0.1、1.0，看 $w, b$ 怎么变。提示：λ 越大，$w$ 越小。
  2. 加一个特征"睡眠时数"，做二维逻辑回归。可视化决策边界。
  3. 故意让两类数据线性可分（比如 X<4 全是 0，X>4 全是 1），看 λ=0 时 $w$ 会不会爆炸。

⑤ 测试你的知识

Q1. 逻辑回归的"逻辑"指的是？

"logistic" 来自 logit = log-odds。我们假设 log-odds 是 x 的线性函数，反解出 sigmoid。这是名字真正的来源。

Q2. Sigmoid 函数 $\sigma(z) = 1/(1+e^{-z})$ 在 $z=0$ 时的值是？

σ(0) = 1/(1+e⁰) = 1/(1+1) = 0.5。这就是决策边界所在 —— 最不确定的点。

Q3. 为什么逻辑回归不用 MSE 损失？

技术上 MSE 也能"硬用"，但有两个数学灾难：损失非凸（卡局部极小）和梯度消失（自信地错时学不动）。对数损失同时解决这两个问题。

Q4. 真实标签 y=1，模型预测 p=0.01（自信地错）。对数损失是多少？

y=1 时损失 = −ln(p) = −ln(0.01) ≈ 4.6。同样情况下 MSE 损失只有 (1−0.01)² ≈ 0.98，对数损失重罚 4.7 倍 —— 这就是"自信地错→损失爆炸"的具体数字。

Q5. 逻辑回归 + 对数损失，单样本梯度对 w 是？

对数损失对 p 求导带来的 1/(p(1−p)) 和 sigmoid 求导带来的 p(1−p) 正好抵消，最终得到干净的 (p−y)·x —— 和线性回归长得一模一样。

Q6. 数据线性可分时，不加正则化的逻辑回归会发生什么？

−ln(p) 永远能通过把 p 推得更接近 0 或 1 来减小，于是 |w| → ∞。所以线性可分的场景必须加正则化。

Q7. sklearn 的 LogisticRegression 里 C=0.01 意味着？

sklearn 用 C 而不是 λ，且 C = 1/λ。C 越小正则化越强，是初学者常见的坑。

Q8. L1 正则化相比 L2 的特点？

L1 用绝对值惩罚，几何上导致最优解常常在坐标轴上（部分维度=0）→ 等于自动选特征。L2 用平方惩罚，只是平滑收缩，不会精确为 0。两者损失都是凸的。

⑥ 后续步骤

立即可学的扩展

  • Softmax 回归（多分类）：3 类以上时，把 sigmoid 换成 softmax $p_k = e^{z_k} / \sum_j e^{z_j}$，损失换成多类交叉熵。逻辑回归是它的二分类特例。
  • 非线性决策边界：手动加 $x_1^2, x_1 x_2$ 等特征 → 多项式逻辑回归，决策边界可以是曲线。
  • SVM、决策树、随机森林：其他分类器，思路不同但解决的是同一类问题。

从逻辑回归到神经网络

逻辑回归 = 单个神经元（带 sigmoid 激活）。把多个这样的神经元堆成"层"，层与层之间嵌套，就是神经网络。最后一层做二分类时，仍然是 sigmoid + 对数损失。所以你已经学会了深度学习里最后一公里的关键组件。

实战里你还会遇到

  • 类别不平衡：违约样本可能只占 5%。需要采样、调整类别权重、用 AUC 而非准确率评估。
  • 评估指标：准确率 / 精确率 / 召回率 / F1 / AUC-ROC，每个适用场景不同。
  • 校准（calibration）：sigmoid 输出的"概率"是真实概率吗？不一定。需要 Platt scaling 或 isotonic regression。
  • 多分类的另一种做法：one-vs-rest（训练 K 个二分类器）vs softmax（一次性输出 K 类概率）。

推荐学习路径

  1. 把第 ④ 节的代码跑一遍，改 λ 玩
  2. 找 kaggle 的 Titanic 数据集做一遍（最经典的二分类入门）
  3. 学 softmax 和多类交叉熵
  4. 学评估指标（混淆矩阵、ROC 曲线）
  5. 读简单神经网络（一两层），你会发现没新东西
